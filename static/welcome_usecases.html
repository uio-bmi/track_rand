Note on GSuite Use-cases:

The power of the GSuite HyperBrowser is best demonstrated by use-cases that show how tools are combined to execute a full analysis scenario. We provide four use-cases that cover different analysis scenarios:

  • The main use-case understanding the genomic distribution of somatic mutations - Performs a high-throughput and fully reproducible analysis of the distribution of non-coding somatic cancer mutation data across cancer types from ICGC against DNase1 hypersensitive sites across cells from ENCODE.
  • Finding the frequency with which a given TF binds to enhancer elements - Answers the question of how often does a given TF (Nanog in these examples) bind enhancer elements in K562 human cells. The use-case has been designed in such a way that the user must get the data from an external source (in this case, UCSC-ENCODE), upload it to History, and process it from there.
  • Inspecting overlap or lack of overlap for SNP microrray platforms - Evaluates a few selected SNP-microarray platforms based on publicly available annotation. The GSuite is manually created and imported into HyperBrowser. The referenced tracks are then downloaded and preprocessed. The Venn Diagram tool is used to calcualte and visualize the overlap between the tracks in the collection.
  • Comparing miRNA targets with DNA binding proteins, somatic mutations and repeating elements - Compiles reference genomic track collections of somatic mutations, repeating elements, histone modifications and DNA binding proteins, from ICGC, ENCODE and from files in a local history. Then uses these reference collections to characterize a set of regions of interest (a single genomic track of miRNA loci).
Note:

This is a specialized version of The Genomic Hyperbrowser, focusing on new functionality for analyzing collections of genomic track. More information on the general HyperBrowser functionality and project can be found on the main version of the HyperBrowser.
